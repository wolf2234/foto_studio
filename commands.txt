-S, --save: Пакет будет отображается в ваших dependencies;
-D, --save-dev: пакет будет отображаться в ваших devDependencies;

1) Установка gulp глобально:
    - npm i gulp-cli -g;

2) Создать файл package.json:
    - npm init;

3) Установить новый пакет:
    - npm i <packname> -D;

3.1) Установить новый пакет:
    - npm install --save-dev <packname>;

4) Установить все пакеты через package.json:
    - npm i;

5) Запустить зборку галп в режиме разработки (dev):
    - npm run dev;

6) Когда здаем заказчику проект. Запуск сборки проекта:
    - npm run build;

7) nvm (Node Version Manager) — это довольно простой скрипт, который позволяет устанавливать, переключать и удалять версии Node.js на лету. 
Проще говоря, nvm даёт вам возможность держать на одной машине любое количество версий Node.js. 
При установке новой версии для неё создаётся отдельная директория, например, 5.0.0 или 4.2.2. 
При переключении версий скрипт подменяет путь до Node.js в PATH.:
    - nvm list;
    - nvm install 0.0.0;
    - nvm use 0.0.0;

8) Отправляем заказчику только папку dist.

9) Возможные ошибки из-за:
- Проблема: img; Решение: (html: "../../img" -> "img", scss: "../../img" -> "../img/");
- Проблема: Шрифты;
